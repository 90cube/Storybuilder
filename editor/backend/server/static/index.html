엔티티 · 행 · 열, 그리고 인물 스키마 구조

1. 기본 개념

용어뜻예
엔티티 (Entity)하나의 대상 — 인물·장소·아이템·사건…"眞 소드마스터"
행 (Row)엔티티 1개 = 표의 가로 한 줄소드마스터 한 줄
열 (Column)모든 엔티티가 공통으로 갖는 칸(세로)이름·타입·요약…
타입 (서브타입)엔티티의 종류인물·장소·아이템

한 줄(행) = 캐릭터 하나. 그 줄에 여러 칸(열)이 있고, 칸마다 값이 들어갑니다. 같은 칸 구성의 줄이 세로로 쌓이면 표(entities 테이블)예요.

2. entities 표의 고정 열 (모든 엔티티 공통)

id name (이름) type (타입) tags (태그) data (타입별 필드 ▼) provenance (출처) review_needed (검토) version (버전) updated_at / updated_by

3. 인물 하위 항목은 어디에? → data 열(JSON) 안

타입마다 칸이 다르므로, 타입별 필드는 data라는 JSON 열 한 칸에 모아 넣습니다. 그래서 새 타입을 늘려도 표 구조(열)는 안 바뀝니다. "인물"의 필드는 schema/character.json이 규정해요.

필드 키이름형식필수
summary한줄요약글✔
personality_traits성격목록—
speech_style말투글—
mbtiMBTI글자—
traits특성목록—

dfu_id / merged_ids식별·병합글자·목록—
images관련 이미지목록—

4. 엔티티에 "달라붙는" 것 (별도 표)

표역할
relations관계 (A —관계어→ B, 양방향)
timeline시점별 상태 변화(역사) 누적
secrets비밀 — 누가 아는가 / 언제 공개되는가

화면의 "한 장(카드)"은 = entities의 행 1개 + 거기 달린 relations·timeline·secrets를 합쳐서 보여주는 것입니다. 저장은 표별로 쪼개고, 보기는 한 장으로 합칩니다.
