# 파이프라인 재설계 + 버전 히스토리 (설계 스펙) — 검토용

> 목적: 한 화(chapter)가 **초안 → 다듬기 → 완성본 → 정사(엔티티/타임라인) → 화 확정**으로 **직선 진행**되게 만들고,
> "잘못 다듬을까봐"의 안전망을 **draft 덮어쓰기 대신 버전 히스토리(분기·롤백)** 로 제공한다.
> 상태: **초안(사용자 검토 대기)**. 승인 후 writing-plans로 태스크 계획 작성.

---

## 0. 현재 문제 (왜 파이프라인이 안 도나)
1. **draft 중심 맴돎**: `/gen`이 모드 무관하게 입력=`draft`, 결과는 kind로 저장되지만 에디터는 `draft`만 표시 → 완성본을 만들어도 안 보이고, 다음 "다듬기"는 또 원래 초안을 다듬음. (채택해야 draft가 바뀜)
2. **단계 버튼이 흩어짐/없음**: 캐릭터감지·정사추출(우측 레일), 승격(정사 패널), 인과로추가(분석 패널). `CTX_RESET_A/B·DB_WRITE·DB_SYNC·DB_SYNC2·PARTIAL_POLISH`는 **버튼 없음**(FSM 명세로만).
3. **엔티티 추출 경로 2개 혼재**: ①집필 중 `분석→인과로 추가`(draft_auto·가안) ②확정 후 `정사 추출→승격`(canon·확정). 역할 경계 불명확.
4. **롤백 없음**: 잘못 다듬어도 되돌릴 수 없음 → 그래서 draft에 "채택"하는 안전 우회가 생김 → 직선 흐름을 막음.

## 1. 핵심 아이디어
**안전망을 버전 히스토리로 옮기면, 파이프라인을 직선으로 펼 수 있다.** 각 생성/편집은 "현재 본문(head)"에 적용되어 **버전 노드**를 만들고, 언제든 이전 노드로 **롤백**하거나 거기서 **분기**할 수 있다. 에디터는 항상 **현재 head(최신본)** 를 보여준다(보류했던 본문모델 #6 해결).

## 2. 재설계된 한 화 파이프라인 (직선 + 분기)
```
① 초안 작성(DRAFT)
② 다듬기 루프(POLISH)  ← 현재 본문 입력, 결과가 새 head
③ 초안 확정            ← "이제 원고로 키운다" 게이트
④ 완성본 생성(EXPAND)  ← 현재 본문 입력, 결과가 새 head
⑤ 원고 확정
⑥ 정사 추출·diff(EXTRACT) ← 현재 head 본문에서 엔티티·관계·사건 + 화 시점 상태 캡처
⑦ 엔티티 반영(PROMOTE) ← 승격: canon 그래프 + 타임라인('현재' 배지)
⑧ 화 확정(SHIP) → 다음 화
```
- **모든 단계 입력 = 현재 head**(draft 고정 X). 부분수정(드래그)은 전 단계에서 가능, 역시 새 head.
- ⑥ 정사 추출이 곧 **엔티티 추출**(어제 만든 타임라인 자동기록 포함).

## 3. 버전 히스토리 / 분기 (안전망)
**데이터 모델** (신규 `versions` 테이블 권장 — 분기를 위해 parent 필요):
```sql
CREATE TABLE versions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  chapter_id INTEGER NOT NULL,
  parent_id INTEGER,                 -- 분기: 부모 버전
  kind TEXT,                         -- draft|polish|expand|partial|manual|accept
  text TEXT NOT NULL,
  label TEXT,                        -- "완성본 v2" 등 표시명
  created_at TEXT, created_by TEXT
);
-- chapters/pipeline_runs에 head_version_id 포인터 추가
```
- 생성/부분수정/수동저장 = **새 노드(parent=head) + head 이동**. 같은 부모에서 다시 생성하면 **형제(분기)**.
- 에디터는 head의 text를 표시. `manuscripts`는 단계 산출물(레거시)로 두거나 versions로 흡수(결정 4).

**UX** (여기가 "약간 어려운" 부분):
- 에디터 상단 또는 우측에 **버전 타임라인** 패널.
- **MVP(1차)**: 최근순 **선형 리스트** — 각 항목 `[kind 아이콘] 라벨 · 시각 · 글자수`. hover=미리보기 팝오버, 클릭=**이 버전으로 되돌리기**(head 이동), "여기서 다시 생성"=분기.
  - 분기는 같은 부모를 가진 형제를 들여쓰기/색으로 구분 표시.
- **2차(선택)**: 미니 **트리 그래프** 시각화(노드·엣지). 풀 git식.
- 안전장치: 되돌리기는 파괴적이지 않음(노드 삭제 X, head만 이동) → 언제든 앞으로/다른 분기로.

## 4. 엔티티 추출 통일 (경로 2개 → 역할 분리)
| 경로 | 시점 | 티어 | 용도 |
|------|------|------|------|
| 집필 중 **분석** | 초안 작성 중(선택) | `draft_auto`(가안) | 인과 캔버스 실시간 미리보기. 비확정. |
| **정사 추출·diff → 승격** | 원고 확정 후(본선) | `canon`(확정) | 그래프 반영 + 타임라인 '현재'. 파이프라인 ⑥⑦ 단계. |
- 즉 둘 다 유지하되 **"가안 미리보기" vs "확정 반영"** 으로 명확히 라벨링. 직선 파이프라인의 정식 단계는 후자.

## 5. 정사화 버튼/단계 정리 (흩어짐·미구현 → 단일화)
- **우측 파이프라인 레일이 "현재 단계의 다음 액션"을 제시**: 현재 head 상태에 맞는 버튼 1~2개만 강조(나머지는 흐림). 단계=액션 1:1.
- 미구현 FSM 상태 흡수:
  - `CTX_RESET_A/B` → EXTRACT **내부 동작**(LLM 컨텍스트 초기화). 버튼 제거.
  - `DB_WRITE·DB_SYNC·DB_SYNC2` → **승격(PROMOTE)** 한 동작에 합침. 버튼 제거.
  - `PARTIAL_POLISH` → **부분수정(드래그)** 기능이 대체. 별도 상태 제거.
- 사용자가 보는 명시 단계: **DRAFT · POLISH · EXPAND · EXTRACT · PROMOTE · SHIP** (6).

## 6. FSM 슬림화 (제안)
`domain/pipeline.py`의 14상태를 실제 6단계로 슬림화:
`DRAFT → POLISH(루프) → EXPAND → EXTRACT(+타임라인) → PROMOTE → SHIP`.
(컨텍스트 초기화·DB 동기화는 단계 내부 동작.) 백엔드 `/run` 단일출처(이미 구현)와 프런트 스테퍼가 이 6단계를 그대로 표시.

---

## 7. 검토 필요 결정 (당신 검사 포인트)
1. **버전 히스토리 UX**: (A) MVP 선형 리스트+되돌리기/분기 → 나중에 (B) 트리 시각화. **추천: A 먼저.**
2. **FSM**: 14상태 → **6단계로 슬림화**(추천) vs 명세 유지+UI만 슬림.
3. **구현 범위/순서**: 단계적 추천 — **①버전 히스토리(안전망) → ②직선 파이프라인(head 기반 입력+에디터 최신본) → ③FSM 슬림화·버튼 단일화 → ④엔티티 경로 라벨 정리.**
4. **저장 모델**: 신규 `versions` 테이블(분기 parent 필요, **추천**) vs 기존 `manuscripts` 확장.
5. **되돌리기 정책**: head만 이동(비파괴, 추천) vs 이후 분기 잘라내기.

## 8. 영향 범위(개략)
- 백엔드: `versions` 테이블·store, `/gen` 입력을 head 기반으로, `/version` 라우트(목록·되돌리기·분기), FSM 슬림화, `/run` 갱신.
- 프런트: 버전 타임라인 패널(신규), 에디터가 head 표시, BottomBar/PipelineRail 단계 단일화, 엔티티 경로 라벨.
- 행동 변화 큼(현 draft-채택 흐름 대체) → 단계적 + 각 단계 검증.
